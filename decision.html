数据驱动方法在优化决策中的应用

摘要

数据驱动方法通过将预测与决策一体化，彻底革新了传统优化手段。本文系统梳理了三大主流范式 — 序贯优化、端到端学习与直接学习——的理论基础、优势、最新进展与局限，随后，文章从数据方法论、优化方法论与决策应用三个维度展开对比，剖析现存挑战。论文最终提供一份方法论指南，并为学术界与产业界在复杂数据环境下提升决策能力指明未来研究方向。

不确定性处理

不确定性的分类与处理

  • 偶然不确定性（Aleatoric Uncertainty）：数据本身的噪声和随机性，是数据生成过程固有的特性，无法通过获取更多数据来消除。
  • 认知不确定性（Epistemic Uncertainty）：模型知识的不足，来自于模型对数据分布的理解不够，可以通过获取更多数据或使用更好的模型来减少。

不确定性估计方法

  • MC Dropout：在推理时保持 Dropout 激活，通过多次前向传播获得预测分布，从而估计模型的不确定性。这种方法可以近似贝叶斯推理。
  • 模型集成（Ensemble）：训练多个不同初始化或结构的模型，集成它们的预测结果，不同模型的预测差异反映了认知不确定性。

风险感知决策

将估计的不确定性作为风险项传递给规划器或决策系统，使得：

  • 在高不确定性区域，车辆采取更保守的决策（如降低速度、增加安全距离）
  • 在低不确定性区域，系统可以进行更激进的优化
  • 实现了从纯粹的预测任务到风险感知的决策系统的转变

一、三种范式

范式对比表

方法 训练目标 预测–决策关系 理论适用性
SO（序贯优化） 预测损失（如 MSE） 预测输出被当作固定输入，供后续优化环节做决策 适用于具有显式优化形式的系统，但计算开销大，且难以回馈决策信息
E2E（端到端） 决策损失（如后悔值） 预测和决策在同一个学习框架中，预测被优化以最小化最终决策损失 理论上最优，但在实践中面临优化难度大、梯度计算复杂等挑战
DL（直接学习） 决策损失（如后悔值） 直接从原始特征学习决策，跳过显式预测步骤 适用于预测任务不是最终目标的场景，可能更高效但可解释性较差

1. SO - 预测与决策的错配

核心思想

序贯优化将问题分为两个阶段：

  • 第一阶段：训练预测模型 $\hat{y} = f(x)$，目标是最小化预测误差（如 MSE）
  • 第二阶段：使用预测结果 $\hat{y}$ 作为输入，通过优化器求解决策 $z^* = \arg\min_z C(z, \hat{y})$

问题所在

这种方法存在根本性的目标不匹配（loss mismatch）：

  • 预测模型优化的是预测精度，但决策关心的是最终的决策质量
  • 最小化预测误差不一定能最小化决策损失
  • 决策损失对预测误差的敏感性因输入而异，但 SO 方法无法捕捉这一点

数学形式

$$\min_{\theta} \mathbb{E}[\|y - f_\theta(x)\|^2]$$

然后：

$$z^* = \arg\min_z C(z, f_\theta(x))$$

2. E2E - 决策驱动学习

核心思想

端到端学习将预测和决策整合在一个学习框架中，直接优化决策损失：

$$\min_{\theta} \mathbb{E}[C(z^*(f_\theta(x)), y)]$$

其中 $z^*(y_{pred}) = \arg\min_z C(z, y_{pred})$

优势

  • 目标对齐：直接优化最终的决策损失，避免了 SO 的目标不匹配问题
  • 自适应学习：模型学会在对决策影响大的区域更加精准
  • 理论上最优：在足够的表达能力下，E2E 学习能达到最优的决策性能

挑战

  • 优化困难：需要通过优化器的梯度反向传播，这要求优化器可微
  • 计算复杂：每次梯度计算都需要求解一个优化问题
  • 非光滑性：许多实际的优化问题（如 MILP）不可微

3. DL - 直接学习

核心思想

直接学习跳过显式的预测步骤，直接从原始特征学习最优决策：

$$z = g(x)$$

$$\min_{\phi} \mathbb{E}[C(g_\phi(x), y)]$$

优势

  • 计算效率：无需显式预测，推理速度快
  • 端到端优化：直接优化决策损失，避免中间的预测步骤
  • 灵活性：可以处理难以显式建模的决策过程

缺点

  • 可解释性差：难以理解模型学到了什么预测关系
  • 可迁移性弱：模型难以迁移到新的决策问题
  • 需要大量数据：缺少预测任务的归纳偏差

二、E2E 三种方法对比

一句话总结

隐式微分通过优化器的隐函数定理求导；代理损失用可微的损失函数近似决策损失；近似方法用参数化的近似替代精确优化。

方法对比表

方法 核心机制 优点 缺点 适用场景
隐式微分 通过隐函数定理计算 $\frac{dz^*}{dy_{pred}}$ 理论严格，精确可微 计算复杂度高，需要求解线性系统 凸优化问题，对精度要求高
代理损失（SPO+） 设计可微损失函数，近似决策损失 实现简单，计算高效 可能存在近似误差，需要精心设计 线性优化问题，快速原型开发
近似方法 用参数化模型 $\tilde{z}(y_{pred})$ 近似 $z^*(y_{pred})$ 灵活，可处理非凸问题 需要额外的预训练，近似精度难以保证 复杂非凸问题，需要快速推理

1 隐式微分（Implicit Differentiation）

原理

利用隐函数定理（Implicit Function Theorem）计算最优解 $z^*$ 对预测值 $y_{pred}$ 的梯度：

$$\frac{dz^*}{dy_{pred}} = -\left(\nabla_z^2 C(z^*, y_{pred})\right)^{-1} \nabla_z \nabla_{y_{pred}} C(z^*, y_{pred})$$

算法流程

  1. 前向传播：给定预测 $y_{pred}$，求解最优决策 $z^* = \arg\min_z C(z, y_{pred})$
  2. 计算海森矩阵：$H = \nabla_z^2 C(z^*, y_{pred})$
  3. 求解线性系统：$Hp = \nabla_z \nabla_{y_{pred}} C(z^*, y_{pred})$
  4. 反向传播：使用 $\frac{dz^*}{dy_{pred}} = -p$ 计算梯度

优缺点

  • 优点：理论严格，精确可微，无近似误差
  • 缺点：
    • 计算复杂度高，需要求解线性系统
    • 要求海森矩阵可逆（即优化问题严格凸）
    • 对于大规模问题可能不可行

代码示例

import torch import torch.nn as nn class ImplicitDifferentiationLayer(nn.Module): def __init__(self, optimizer_fn): super().__init__() self.optimizer_fn = optimizer_fn def forward(self, y_pred): # 求解最优决策 z* z_star = self.optimizer_fn(y_pred) return z_star def backward(self, grad_output): # 使用隐函数定理计算梯度 # 这里需要计算海森矩阵和线性系统求解 # 实现细节取决于具体的优化问题 pass

2 代理损失（Surrogate Loss，以 SPO+ 为代表）

原理

SPO+（Shortest Path Operator+）为线性优化问题设计了一个可微的代理损失函数。对于线性优化问题：

$$z^*(y_{pred}) = \arg\min_z c(y_{pred})^T z$$

SPO+ 定义损失函数为：

$$L_{SPO+}(y_{pred}, y_{true}) = c(y_{pred})^T z^*(y_{true}) - c(y_{true})^T z^*(y_{true})$$

其中 $z^*(y_{true})$ 是在真实成本下的最优解。

直观理解

SPO+ 损失衡量的是：使用预测成本 $c(y_{pred})$ 求得的最优解在真实成本 $c(y_{true})$ 下的性能与真实最优解的差距。

优缺点

  • 优点：
    • 实现简单，只需求解两个线性优化问题
    • 计算高效，适合大规模问题
    • 有理论保证（在某些条件下）
  • 缺点：
    • 仅适用于线性优化问题
    • 可能存在近似误差
    • 需要能够求解优化问题

代码示例

import torch import torch.nn as nn class SPOPlusLayer(nn.Module): def __init__(self, solver): super().__init__() self.solver = solver # 线性优化求解器 def forward(self, y_pred, y_true): # 求解在预测成本下的最优解 z_pred = self.solver(y_pred) # 求解在真实成本下的最优解 z_true = self.solver(y_true) # SPO+ 损失 loss = torch.sum(y_pred * z_true) - torch.sum(y_true * z_true) return loss, z_pred

3 近似方法（Approximation Methods）

原理

使用一个参数化的神经网络 $\tilde{z}_\phi(y_{pred})$ 来近似最优决策函数 $z^*(y_{pred})$：

$$\tilde{z}_\phi(y_{pred}) \approx z^*(y_{pred})$$

学习目标是最小化：

$$\min_{\phi} \mathbb{E}[\|z^*(y_{pred}) - \tilde{z}_\phi(y_{pred})\|^2]$$

两阶段训练

  1. 第一阶段：预训练近似函数
    • 生成训练数据：对多个 $y_{pred}$ 值求解 $z^*(y_{pred})$
    • 训练神经网络 $\tilde{z}_\phi$ 来拟合这些解
  2. 第二阶段：端到端微调
    • 将 $\tilde{z}_\phi$ 作为可微的优化器
    • 通过决策损失反向传播来调整预测模型

优缺点

  • 优点：
    • 灵活性高，可处理任意类型的优化问题（包括非凸、整数规划）
    • 推理速度快，只需一次前向传播
    • 可以利用深度学习的表达能力
  • 缺点：
    • 需要大量的预训练数据
    • 近似精度难以保证
    • 需要两个训练阶段，工程复杂度高

代码示例

import torch import torch.nn as nn class ApproximationOptimizer(nn.Module): def __init__(self, input_dim, output_dim, hidden_dim=128): super().__init__() self.net = nn.Sequential( nn.Linear(input_dim, hidden_dim), nn.ReLU(), nn.Linear(hidden_dim, hidden_dim), nn.ReLU(), nn.Linear(hidden_dim, output_dim) ) def forward(self, y_pred): return self.net(y_pred) # 预训练阶段 def pretrain_approximator(solver, train_data, epochs=100): model = ApproximationOptimizer(input_dim=10, output_dim=5) optimizer = torch.optim.Adam(model.parameters()) for epoch in range(epochs): for y_pred in train_data: z_true = solver(y_pred) z_pred = model(y_pred) loss = torch.nn.functional.mse_loss(z_pred, z_true) optimizer.zero_grad() loss.backward() optimizer.step() return model # 使用示例 class PerturbationOptimizer(nn.Module): def __init__(self, solver, sigma=0.05, n_samples=20): super().__init__() self.solver = solver # 你的 MILP 求解器 self.sigma = sigma # 噪声标准差 self.n_samples = n_samples # 扰动样本数 def forward(self, y_pred): # 添加噪声，求解多次 z_samples = [] for _ in range(self.n_samples): y_noisy = y_pred + torch.randn_like(y_pred) * self.sigma z = self.solver(y_noisy) z_samples.append(z) # 返回期望解 return torch.stack(z_samples).mean(dim=0)
