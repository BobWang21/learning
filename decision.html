预测与决策结合的范式

数据驱动方法在优化决策中的应用

摘要

数据驱动方法通过将预测与决策一体化，彻底革新了传统优化手段。本文系统梳理了三大主流范式 — 序贯优化、端到端学习与直接学习——的理论基础、优势、最新进展与局限，随后，文章从数据方法论、优化方法论与决策应用三个维度展开对比，剖析现存挑战。论文最终提供一份方法论指南，并为学术界与产业界在复杂数据环境下提升决策能力指明未来研究方向。

一、三种范式

范式对比表

方法 训练目标 预测–决策关系 理论适用性
SO（序贯优化） 预测损失（如 MSE） 预测输出被当作固定输入，供后续优化环节做决策 适用于具有显式优化形式的系统，但计算开销大，且难以回馈决策信息
E2E（端到端） 决策损失（如后悔值） 预测模型在闭环框架内训练，专注于优化决策表现 适用于要求高决策精度、预测模型易于重训并能与决策目标紧密对齐的任务
DL（直接学习） 决策损失（如任务专用损失函数） 预测模型直接学习最优决策策略，无需依赖显式优化结构 适用于决策问题复杂且缺乏显式优化结构的场景，如自动驾驶系统

1. SO - 预测与决策的错配

在实际应用中，SO 常常因无法达到绝对预测精度而遭遇挑战。预测模型通常采用梯度下降算法训练，一旦预测损失降至预设阈值即停止，以防过拟合，同时尽可能有效地捕捉模式。然而，预测误差的降低并不必然带来决策优势，因为现实场景中预测性能与决策性能之间的关系往往是非对称的。

与基于范数的预测损失所具有的对称结构不同，决策准则极少是直白明确的。以负荷预测为例，电力系统的运行成本具有非对称性：

  • 预测偏高会导致不必要的发电、增加运行备用及资源浪费
  • 预测偏低则可能引发供应不足，迫使其采取昂贵的一次调频、启用快速启动机组，甚至危及系统可靠性

基于范数的统计预测损失对正负误差一视同仁，无法刻画这类运行非对称性。这种失配源于决策过程必须结合现实环境，去捕捉时间序列中更为隐性的模式。

2. E2E - 决策驱动学习

也称为决策驱动学习。该方法将优化结构嵌入训练过程中，通过决策反馈直接调整预测模型，实现从预测到决策的闭环学习，从而更好地对齐预测与决策目标，有助于减少序贯优化方法中存在的偏差问题。

一种直观的决策损失定义，用于量化「在预测场景 ŷ 下做出的决策」与「在真实场景 y 下做出的决策」之间的差异，通常表示为：

L = f(ẑ*; y) - f(z*; y)

其中：

  • f 是优化问题中的目标函数，在给定参数 y 下，采取决策 ẑ* 所对应的成本或收益
  • ẑ* 是最优解，ẑ* = M(ŷ)
  • M 表示从输入预测 ŷ 到最优决策 ẑ* 的映射

E2E求解方法

一般可以通过三类E2E方法求解：

  • 隐函数方法：通过优化问题的KKT条件将上式表示成雅可比矩阵的相应形式
  • 代理损失函数法：以SPO+损失函数为代表，将上式转化为更易求解的SPO+损失函数的梯度形式
  • 近似求解法：通过对预测值叠加高斯噪声ϵ，并计算其期望值，即决策问题的扰动解

3. DL - 直接学习

预测模型直接学习最优决策策略，无需依赖显式优化结构。适用于决策问题复杂且缺乏显式优化结构的场景，如自动驾驶系统。

二、E2E三种方法对比

一句话总结

  • 如果问题是凸优化 + 计算资源充足 → 用 隐式微分
  • 如果问题是线性目标 + 约束与预测无关 → 用 SPO+
  • 如果问题有整数变量、非凸、或约束含预测参数 → 用 近似方法

方法对比表

场景特征 隐式微分 代理损失（SPO+） 近似方法
目标函数类型 凸、可微 线性（必须） 任意
约束是否含预测参数 可以 不可以 可以
决策变量类型 连续 连续（整数需松弛） 连续/整数均可
优化问题类型 凸优化 线性规划 任意（非凸、离散）
数据规模 小-中 大 中-大
计算资源要求 高（GPU集群） 低-中 中
对精度要求 极高 高 中-高

1 隐式微分（Implicit Differentiation）

核心思想

利用 KKT 条件 和 隐函数定理，将优化问题的最优解对预测输入的敏感性，转化为求解一个线性方程组。

适合场景

场景特征 说明
优化问题类型 凸优化、可微问题（二次规划、线性规划）
决策变量类型 连续变量为主
约束形式 可以是等式/不等式约束，无需特殊限制
对精度要求 极高（如金融风控、医疗决策）
计算资源 充足（GPU 集群）

优点

优点 说明
精确梯度 基于解析推导，无近似误差
理论严谨 建立在坚实的凸优化和隐函数定理基础上
可解释性强 保留了完整的优化结构，决策过程透明
处理复杂约束 可处理等式、不等式、凸约束组合

缺点

缺点 说明
计算成本极高 每轮迭代需计算并求逆雅可比矩阵，复杂度 O(n³)
仅限凸问题 非凸问题无法保证 KKT 条件的充分性
无法处理整数变量 离散决策变量不满足可微要求
实现复杂 需要手推 KKT 系统并实现雅可比求解

2 代理损失（Surrogate Loss，以 SPO+ 为代表）

核心思想

尽管隐式微分方法具有诸多优势，但其计算成本仍然是一个关键问题。每次训练迭代都需要计算并求逆雅可比矩阵，这对于大批量输入来说计算量巨大。为了缓解这一问题，使用代理损失函数来简化梯度计算。

对于定义为 f(z, y) = y^T z 的线性目标函数，SPO+ loss 定义如下：

L_SPO+ = -(2ŷ - y)^T M(2ŷ - y) + (2ŷ - y)^T M(y)

适合场景

场景特征 说明
目标函数 必须是线性的：f(z,y) = y^T z
约束条件 必须与预测输入无关：约束中不能出现 y
优化问题类型 线性规划、整数线性规划（需松弛）
数据规模 大规模（百万级样本）
计算资源 有限（普通服务器）

典型应用

  • 电力市场电价预测 + 储能套利
  • 物流配送订单分配
  • 广告预算分配

优点

优点 说明
计算高效 只需两次优化求解，无矩阵求逆
理论保证 贝叶斯风险分析证明与最小化遗憾等价
易于实现 只需包装现有求解器，无需修改求解器内部
可扩展性 适用于百万级数据，计算成本与预测模型相当

缺点

缺点 说明
仅限线性目标 非线性目标需要额外处理（如线性化）
约束必须独立 约束不能含预测参数，否则理论失效
整数变量需松弛 对 MILP 问题需松弛为 LP，可能引入误差
仍需求解优化 每轮迭代仍需求解两次优化，对 NP 难问题仍有挑战

3 近似方法（Approximation Methods）

核心思想

除了计算效率之外，在具有多面体可行域和线性目标函数的优化中，梯度信息的有效性也存在问题，因为 y 的变化可能并不总是影响最优解 z*，并且可能出现无穷多解的情况。

通过向预测值添加噪声，使决策函数平滑化，将原本不连续的极值点解转化为极值点的期望：

z_perturb* = E_ε[M(ỹ + εZ)]

适合场景

场景特征 说明
优化问题类型 非凸、离散、组合优化（最短路径、Top-K、整数规划）
决策变量类型 整数变量、离散选择
目标函数 任意形式（线性/非线性）
约束条件 任意形式（可含预测参数）
计算资源 中等（可接受多次优化求解）

典型应用

  • 最短路径规划（离散选择）
  • 机器人控制（连续动作空间）
  • 组合优化问题（图优化、排序）

优点

优点 说明
适用性最广 可处理非凸、离散、组合优化问题
梯度平滑 通过期望使梯度连续，避免零梯度问题
实现简单 只需包装现有求解器，无需修改
处理整数变量 可直接处理 0-1 变量、离散选择
约束可含预测 约束中的预测参数自动被噪声平滑

缺点

缺点 说明
样本需求高 需要多个扰动样本来稳定梯度（通常 10-50 个）
计算开销大 每轮迭代需求解 n_samples 次优化问题
近似误差 噪声引入的偏差可能使解偏离真值
方差控制难 噪声过大梯度不准，过小梯度不连续

代码示例

class PerturbedOptimizer: def __init__(self, solver, sigma=0.05, n_samples=20): self.solver = solver # 你的 MILP 求解器 self.sigma = sigma # 噪声标准差 self.n_samples = n_samples # 扰动样本数 def forward(self, y_pred): # 添加噪声，求解多次 z_samples = [] for _ in range(self.n_samples): y_noisy = y_pred + torch.randn_like(y_pred) * self.sigma z = self.solver(y_noisy) z_samples.append(z) # 返回期望解 return torch.stack(z_samples).mean(dim=0)
