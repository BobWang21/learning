Post-Training 全景指南

1. 引言：什么是 Post-Training？

LLM的训练通常分为两个大阶段：预训练（Pre-training） 和 后训练（Post-training）。预训练阶段通过海量无标注文本让模型学会语言的基本规律和世界知识，产出的是一个"什么都知道一点、但什么都不太好用"的基座模型。而 Post-training 则是将这个"毛坯房"精装修成真正好用的产品的过程。

Post-training 的三个核心目标：

  • SFT 教模型"说什么"
  • RL 教模型"怎么想"
  • 偏好优化教模型"怎么选"

2. 直观理解

一个餐厅的类比。想象你开了一家餐厅，招了一个天赋异禀的厨师（Pre-trained Model）。这个厨师读过所有的菜谱书（预训练数据），知道各种食材和烹饪技法，但从没真正为客人做过菜。

  • SFT（监督微调） 就像是让资深厨师手把手教他做几道招牌菜
  • RLHF（基于人类反馈的强化学习） 则让食客品尝菜品并排序
  • DPO（直接偏好优化） 是一种更简洁的方式，不需要单独训练评分系统
  • RLVR（基于可验证奖励的 RL） 适用于有标准答案的场景
  • Agentic RL 则是最新的方向：不仅要求厨师会做菜，还要会查菜谱、去市场采购

3. 核心方法详解

3.1 SFT：监督微调

监督微调是 Post-training 最基础也最直观的方法。其核心思路是：收集高质量的 (prompt, response) 数据对，然后用标准的交叉熵损失函数对预训练模型进行微调。

关键认知： SFT 教会模型"输出的格式和风格应该是什么样的"，但本质上是在模仿，无法让模型学会超越训练数据的能力。这就是为什么需要 RL。

3.2 RLHF：基于人类反馈的强化学习

RLHF 是 InstructGPT 和 ChatGPT 背后的核心技术。其完整流程分为三步：

步骤 说明
Step 1：SFT 首先收集人类撰写的高质量回答，对预训练模型进行监督微调
Step 2：训练 Reward Model 让人类标注者对多个回答进行排序，训练一个奖励模型
Step 3：PPO 优化 使用训练好的 Reward Model 作为奖励信号，通过 PPO 算法进行优化

3.3 GRPO：去掉 Critic 的轻量级 RL

GRPO（Group Relative Policy Optimization）由 DeepSeek 团队在 2024 年提出，是当前开源推理模型训练中最流行的 RL 算法。GRPO 的核心创新在于用组内相对排名来替代 Value Model 估计 advantage，从而去掉 Critic 模型。

维度 PPO GRPO
所需模型数 4 个 2~3 个（无 Critic）
显存需求 高 较低
训练稳定性 需要精细调参 相对简单
典型应用 ChatGPT, InstructGPT DeepSeek-R1, Qwen3

3.4 RLVR：可验证奖励

RLVR（Reinforcement Learning with Verifiable Rewards）是 2025 年最重要的技术趋势之一。DeepSeek-R1 就是使用 GRPO + RLVR 训练的典型代表。

RLVR 的奖励通常包含两部分：

  • 准确性奖励（答案是否正确）
  • 格式奖励（输出是否符合要求的格式）

3.5 DPO 及其变体

DPO（Direct Preference Optimization）提供了一种完全不同的思路：直接从偏好数据中优化策略，不需要训练 Reward Model，也不需要 RL 训练循环。

DPO 的核心洞察是：最优奖励函数与最优策略之间存在解析关系，因此无需单独建模奖励函数，直接优化策略即可。

方法 核心改进 适用场景
DPO 用分类损失替代 RL 通用对齐
SimPO 移除 reference log-ratio，梯度更稳定 噪声数据、众包标注
ORPO 在 odds-space 中优化，处理类别不平衡 多语言、长尾数据
KTO 基于前景理论的不对称损失 高风险领域（法律、医疗）

4. DeepSeek-R1：纯 RL 训练推理模型的里程碑

DeepSeek-R1 是 2025 年初最具影响力的工作之一，它首次证明了 纯 RL 训练（不需要 SFT）就能让模型涌现出强大的推理能力。

DeepSeek-R1 的训练分为两条路线：

  • R1-Zero（纯 RL 路线）：直接在预训练的 DeepSeek-V3 基座模型上，使用 GRPO + RLVR 进行训练，完全跳过 SFT 阶段
  • R1（完整路线）：在 R1-Zero 的基础上，加入了 SFT 数据进行冷启动，然后再进行 RL 训练

5. 总结与关键要点

  • 第一，SFT 是基础但不够。 SFT 教会模型输出的格式和风格，但无法让模型学会超越训练数据的推理能力
  • 第二，RL 是提升推理能力的关键。 从 PPO 到 GRPO，RL 算法在不断简化和高效化
  • 第三，奖励信号的设计至关重要。 从 RLHF 到 RLVR，奖励信号的获取方式在不断演进
  • 第四，Online RL vs. Offline Preference Optimization 各有所长。 DPO 等 offline 方法简单稳定，适合偏好对齐；PPO/GRPO 等 online 方法更适合提升推理能力
  • 第五，Agentic RL 是下一个前沿。 从单轮问答到多轮工具使用，Post-training 正在向训练完整的"智能体"方向发展
